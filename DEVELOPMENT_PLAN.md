# WordVision 开发计划

本文档用于约定 WordVision：AI 视觉词汇学习助手的版本管理、MVP 范围、迭代方向、验收标准和需要用户准备或确认的事项。

## 版本管理策略

### 分支策略

- `main`：稳定分支，只合入已验收或可交付的版本。
- `develop`：日常集成分支，用于合并功能开发结果。
- `feature/*`：功能分支，例如 `feature/quiz-flow`、`feature/mistake-book`。
- `fix/*`：问题修复分支，例如 `fix/storage-migration`。
- `docs/*`：文档分支，例如 `docs/delivery-notes`。

### 版本号策略

建议使用语义化版本：

- `0.1.0`：MVP，可完成核心本地学习流程。
- `0.2.0`：补充错题本、统计、收藏、详情弹窗和深色模式体验。
- `0.3.0`：增强 AI 辅助内容、发音、词库管理和复习策略。
- `1.0.0`：功能稳定、体验完整、文档和验收材料齐备，可作为正式演示版本。

### 提交规范

建议采用简洁的 Conventional Commits：

- `feat:` 新功能。
- `fix:` 问题修复。
- `docs:` 文档更新。
- `style:` 样式调整，不改变业务逻辑。
- `refactor:` 重构，不改变外部行为。
- `test:` 测试相关。
- `chore:` 构建、依赖或工程配置。

示例：

```text
feat: add four-option vocabulary quiz
docs: add delivery README and development plan
fix: persist theme preference in localStorage
```

## MVP/迭代计划

### MVP：本地核心学习闭环

目标：用户可以在本地完成一次完整词汇学习流程，并保留基础学习记录。

范围：

- 首页入口和基础导航。
- 词库选择。
- 词汇总览。
- 学习设置。
- 四选一学习流程。
- 学习总结。
- localStorage 保存学习进度和结果。
- 基础响应式布局。

建议版本：`0.1.0`

### 迭代 1：复习与个性化

目标：让用户可以围绕错题、收藏和偏好进行复习。

范围：

- 错题本。
- 收藏词汇。
- 详情弹窗。
- 深色模式。
- 发音能力。
- 更完整的学习设置持久化。

建议版本：`0.2.0`

### 迭代 2：统计与学习反馈

目标：让用户看到长期学习效果，并获得下一步学习建议。

范围：

- 学习统计页面。
- 正确率、累计学习数、错题趋势和词库完成度。
- 总结页的复习建议。
- 本地数据初始化、迁移和清理能力。
- 关键工具函数测试。

建议版本：`0.3.0`

### 迭代 3：AI 视觉学习增强

目标：增强 AI 视觉联想和内容生成能力，让产品名中的“AI 视觉词汇学习助手”具备可感知价值。

范围：

- AI 生成视觉联想描述。
- AI 辅助例句和释义。
- AI 生成或推荐记忆图片。
- AI 生成四选一干扰项。
- API 调用失败时降级到本地静态数据。
- 明确用户数据和 API Key 的安全边界。

建议版本：`0.4.0`

### 正式版：演示与交付完善

目标：形成稳定、可演示、可维护的前端应用。

范围：

- 完整页面状态处理，包括空状态、加载态、错误态。
- 主要浏览器兼容验证。
- README、开发计划、数据说明和演示脚本完善。
- 基础端到端流程验证。
- 生产构建通过。

建议版本：`1.0.0`

## 验收标准

### 功能验收

- 用户可以从首页进入词库选择，并选择一个词库开始学习。
- 用户可以查看词汇总览，并打开词汇详情弹窗。
- 用户可以配置学习数量、题目顺序、自动发音、深色模式等学习偏好。
- 用户可以完成四选一学习，每题能判断正确或错误。
- 学习结束后可以看到总结，包括正确率、答题数量、错题列表和复习建议。
- 答错的词汇会进入错题本，后续可以再次练习。
- 用户可以收藏或取消收藏词汇。
- 统计页能展示累计学习数据和基础正确率。
- 刷新页面后，localStorage 中的学习记录、设置、收藏、错题和主题偏好仍然保留。
- 发音功能在支持 Web Speech API 的浏览器中可用；不支持时应给出可接受的降级体验。

### 界面与体验验收

- 应用在桌面端和常见移动端宽度下布局可用。
- 深色模式切换后主要页面文字、按钮和卡片对比度清晰。
- 四选一选项点击区域足够明确，并能区分选中、正确和错误状态。
- 详情弹窗可打开、关闭，并不会遮挡关键操作。
- 页面无明显控制台错误。

### 工程验收

- `npm install` 可以完成依赖安装。
- `npm run dev` 可以启动本地开发服务。
- `npm run build` 可以完成生产构建。
- localStorage 读写逻辑有统一封装，避免页面直接散落存储 key。
- 词库数据结构清晰，至少包含单词、释义、词性、例句、音标、难度或标签等字段。
- 关键学习逻辑与 UI 展示逻辑适度分离，便于后续测试。

### 文档验收

- README 包含项目简介、功能介绍、安装运行方法、AI 在项目中的作用、项目文件结构和后续改进方向。
- DEVELOPMENT_PLAN 包含版本管理策略、MVP/迭代计划、验收标准和需要用户准备/确认的事项。
- 文档内容与本地 React + Vite + Tailwind CSS 应用定位一致。

## 需要用户准备/确认的事项

- 词库来源：确认首批词库内容、授权状态、字段格式和是否允许内置到前端代码中。
- 目标用户：确认是面向学生、考试备考者、儿童启蒙、成人自学，还是课堂演示。
- 学习语言：确认释义以中文为主、英文为主，还是中英双语。
- AI 接入方式：确认 MVP 是否只使用静态数据模拟 AI 效果，或需要接入具体 AI API。
- 视觉素材：确认是否需要真实图片、AI 生成图片、图标化插画，或只提供文字视觉联想。
- 发音方案：确认优先使用浏览器 Web Speech API，还是准备本地音频文件。
- 数据持久化边界：确认是否只使用 localStorage，还是后续需要账号登录和云端同步。
- 统计维度：确认需要展示哪些学习指标，例如天数、正确率、词库进度、错题变化、收藏数量等。
- 设计风格：确认默认主题、品牌色、深色模式要求和移动端优先级。
- 浏览器范围：确认需要支持的浏览器版本，尤其是发音和 localStorage 行为。
- 演示要求：确认是否需要准备演示数据、演示流程、截图或打包部署版本。
